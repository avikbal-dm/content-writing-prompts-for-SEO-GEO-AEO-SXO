# GLOSSARY

Every term the engine uses, defined once. No cryptic shorthand survives outside this file.

## TIERS

**PILLAR**
A primary landing or hub page meant to own a topic. Runs the full engine: files 01 through 04. Maximum depth, maximum internal linking.

**CLUSTER (supporting)**
A narrower page that supports a pillar. Runs `lite-path.md` only. Same brand and quality rules, proportionate effort. Never gets the full 17-step treatment.

## DEPTH MODEL

Depth is set by competitive reality, not by a fixed word count. Three levels:

**Standard depth**
Covers the primary intent and the obvious follow-up questions. Typical for a focused cluster page.

**Deep depth**
Covers primary intent, secondary intents, the full entity set, and the objections that delay a decision.

**Maximum depth**
Exhaustive intent and entity coverage. Every reasonable subtopic a buyer or AI engine would expect, with zero padding. This is the target for PILLAR pages. The old "L99" label meant this. Word count follows from coverage, never the reverse.

## INTERNAL LINKING FLOOR

**PILLAR:** high contextual density. Target 50 or more internal links across the cluster, every one pointing to a real existing URL or a planned URL flagged as planned. Density earns topical authority. Volume for its own sake does not.

**CLUSTER:** 3 to 8 contextual internal links to real existing URLs, including one link up to the parent pillar.

## STRUCTURAL TERMS

**Answer capsule**
A 40 to 60 word direct answer placed immediately under the H1, written to be quotable by an AI engine and to win a featured snippet. Self-contained. No "as we will see" hedging.

**Page existence test**
The gate in Phase 0. A page may exist only if it owns a defensible angle no top-10 result occupies. No defensible angle means no page.

**Cannibalisation gate**
The Pre-Flight check. Before planning anything, confirm no existing page on the domain already targets this keyword cluster. The decision is NEW, CONSOLIDATE, or REWRITE.

**Fast win**
A keyword where weak competitors currently rank, so the page can realistically place quickly. Prioritised by speed, not just volume.

## DATA INTEGRITY TERMS

**Fetch-first**
Pull the live number or page before you describe it. Never characterise a competitor, a SERP feature, or a metric from memory.

**No-data fallback**
If a tool returns nothing, write "No data returned" and move on. Never fill the gap with an estimate. Three honest gaps beat twelve confident fabrications.

## OPTIMISATION ACRONYMS

**AEO** Answer Engine Optimisation. Structuring content so answer engines can extract and cite it.
**GEO** Generative Engine Optimisation. Earning visibility inside AI Overviews and chat answers.
**AIO** AI Overview. Google's generated answer block at the top of some SERPs.
**PAA** People Also Ask. The expandable question block on a SERP.
**SXO** Search Experience Optimisation. Matching page type and experience to what the SERP rewards.
**E-E-A-T** Experience, Expertise, Authoritativeness, Trust. Google's quality signal set.

## STATE PASSING

Each stage produces an output the next stage consumes. Save each stage's output before moving on. Stage 02 expects the research artifact from 01. Stage 03 expects both. The quality gate reviews the full brief. Treat the outputs as files, not as memory.
